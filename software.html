Roadmap

This is a roadmap of the different stages of maturity of project ATHN and what software will be most important to make in those stages. We're currently heading into the first stage, the draft stage.

  1. The draft stage: Very unstable standards. The most important thing right now is to get enough people (primarily technical) using the project that we can develop the standards to a point where they're good enough to progress to the next stage. This stage is about moving fast, trying things out, and finding out what works and what doesnt, our attention should be mostly on the standards.

    2. Software needed: A usable browser, a usable server

  2. The popularization stage: Standards are still not stable, but mostly in place. This stage is about scaling, getting the more average users to start using ATHN. And about polishing the standards with the experiences we get from that, this is the stage where the base standards are finalized. One crucial piece of software needed to progress to this stage is an ATHN to web proxy, this is neccessary to allow people to switch to ATHN (with all the benefits) without losing the audience that is the web. Without this proxy it wouldnt be worth it or feasable for most people to make the switch because of the loss of audience.

    3. Software needed: A basic ATHN-web proxy, a usable browser for every major platform, usable servers for every major use case

  3. The release stage: First stable release of the base standards, from now on they wont have any breaking changes. It's possible that we could be working on companion standards at this stage. This stage is about developing very stable, easy to use and high quality software for ATHN. But also about finding out what we can do with this now stable project. It's also a time for growth because at this stage it should be safe and easy for anyone to get on board

    4. Software needed: A fully featured ATHN-web proxy (both software and service), A free (for basic use cases) ATHN site hosting service, rock solid browsers for every major platform, rock solid servers for every major use case, at least one goodd document validation client
